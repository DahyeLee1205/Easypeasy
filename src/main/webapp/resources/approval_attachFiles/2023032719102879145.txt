- 사용자(빛나)
1. 회원가입 닉네임 추가 (중복확인)
2. 메인페이지 - 고객센터 - 업체등록 선택 화면 필요

- 검색(한수)
1. 업체등록 페이지에서 사업자등록번호, 이메일, 업체명, 주소(주소입력 프로그램) 업태, 주차 체크박스로 여부 확인, 사진 업로드(예약 페이지 업체정보 참조) 필요
    웹사이트 삭제,  
2. 음식점 선택 후 세부화면 (태진)

- 예약 및 결제(태진)
1. 전체 UI 통일
2. 업체정보 삭제
3. 총 결제금액 적립금 사용 여부에 따라 변경
4. 메뉴 선택에서 삭제
5. 예약 결제 2번째거 삭제 (간단 내용)

- 업체(라현)
1. 업체 메인페이지에서 예약 현황 달력/시간표 보여주기
2. 사용자가 보는 업체 페이지 : 문의사항 작성 (음식점 페이지에 문의하기 추가)
3. 예약 관리 페이지 : 더보기 -> 상세정보로 변경
   예약확인 상태변경 버튼 추가
   게시판에 결제된 금액 추가
   검색 카테고리별로 키워드 검색 기능 구현
4. 정산 페이지 추가
5. 매장관리 달력설정 : 예약가능/불가능 변경과 일별 예약 현황 구분이 명확하게 되도록
   문의답변 (사용자 문의사항 받는 게시판)
6. 고객센터 : 관리자에게 하는 문의요청 1대1 문의, FAQ, 문의사항 게시판 (업체 입장에서 하나의 게시판으로 보이게)
7. 업체 첫로그인 하자마자 업체 상품 등록 화면 넣고 다른 기능 접속할 수 있도록
   업체 썸네일 이미지 넣기, 메뉴 파트(사진, 메뉴 이름, 메뉴 금액, 메뉴 설명-메뉴 설명은 업체 선택 영역)
8. 업체가 리뷰에 좋아요 누르기 기능

- 관리자(빛나)
1. 로그인 페이지 : 아이디 비밀번호 찾기 삭제
2. 회원조회 탭 삭제(바로 회원조회 페이지로)
3. 회원관리 페이지 : 상태(정상, 이용정지, 탈퇴) 별 조회 카테고리 기능 구현
   하단에 회원 상태별 현황 보여주기
   회원 상세 정보 회원가입과 동일하게 맞추기
   회원 상세 정보 수정 완료 직전에 관리자 비밀번호 입력
   회원 전체 조회 페이지에서 탈퇴 처리 하기 전에도 비밀번호 입력
   회원 전체 조회 페이지에서 적립금 가능하도록 기능구현
   회원 전체 조회 페이지에서 모든 회원 전부 선택할 수 있는 버튼 추가
4. 회원 이용내역 : 결제 수단 추가
5. 업체조회 탭 삭제(바로 업체조회 페이지로)
6. 업체조회 페이지 : 이름을 상호명으로 변경
   업체 예약 현황 조회 버튼(업체가 보는 예약 현황을 관리자도 볼 수 있게)
7. 사이트 관리 페이지 : 검색 인덱스 삭제(카테고리랑 동일 내용)
   카테고리 항목 추가 기능구현
8. 고객센터 탭 : 사용자 파트 / 업체 파트 구분해서 나누기
9. 고객센터 페이지 : 
   1대1 문의, FAQ, 공지사항 - 사용자 파트 / 업체 파트 구분
10. 리뷰 관리 페이지 : 리뷰 작성자 적립금 직접 제공하는 부분 삭제, 자동화 (리뷰작성 3일 후)
     리뷰 숨기기 기능 구현
